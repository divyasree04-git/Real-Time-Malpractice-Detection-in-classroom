                        WHAT WAS THE PROBLEM?
???????????????????????????????????????????????????????????????????????????????

The previous implementation had threading locks and lazy initialization that 
caused these issues:

1. ? Global variable assignment inside lock
   • Line: with camera_lock: cap = None
   • This was causing global scope issues
   • Camera initialization was delayed/blocked

2. ? Complex lazy initialization
   • Camera tried to initialize only on demand
   • Multiple initialization attempts conflicted
   • Lock prevented proper initialization

3. ? Threading lock overhead
   • camera_lock was causing delays
   • Multiple acquire/release cycles
   • Unnecessary complexity for single camera

4. ? get_camera() function complexity
   • Added extra layer of indirection
   • Made debugging difficult
   • Return value handling unclear


???????????????????????????????????????????????????????????????????????????????
                         THE SOLUTION APPLIED
???????????????????????????????????????????????????????????????????????????????

REVERTED TO SIMPLE, RELIABLE APPROACH:

Old Code (Problematic):
??????????????????????????????????????????????????????????????????????????
cap = None
camera_lock = threading.Lock()

def init_camera():
    global cap
    try:
        cap = cv2.VideoCapture(0)
        if not cap.isOpened():
            cap = None
            return False
        return True
    except:
        cap = None
        return False

def get_camera():
    global cap
    if cap is None:
        init_camera()
    return cap

New Code (Working):
??????????????????????????????????????????????????????????????????????????
cap = cv2.VideoCapture(0)  # Simple and direct!

# That's it! No locks, no lazy init, no complexity.
